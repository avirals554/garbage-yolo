How it works

Four moving parts wired together so frames flow with low latency.

1
Browser captures the webcam

getUserMedia() grabs a video track from the camera. The user clicks Start so the browser permits it.

2
WebRTC peer connection

Browser and Python server each create an RTCPeerConnection and exchange a single SDP message via POST /offer. After that, frames flow peer-to-peer — no HTTP per frame.

3
Server runs YOLOv8

An aiortc media track wraps the incoming video, runs YOLO inference on each frame, and draws bounding boxes over the detections.

4
Annotated stream comes back

The processed track is added to the same peer connection. The browser's ontrack event fires and the video element shows the annotated feed.

Stack

aiohttp serves the site and the signaling endpoint. aiortc handles WebRTC on the Python side. ultralytics runs YOLOv8 inference. The frontend is plain HTML, CSS, and JavaScript — no frameworks.

Run it locally

Start the server with python project/server.py and open http://127.0.0.1:8080. Camera access requires a secure context, which localhost already counts as.
